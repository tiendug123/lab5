Chứng thực người dùng REST API với JSON Web Token (JWT):

Để thực hiện các lời gọi REST API, đầu tiên người dùng cần chứng thực với server thông qua username/password. Nếu hợp lệ, server sẽ trả về một JSON Web Token (JWT) có giá trị trong một thời gian nhất định (ví dụ như 24 giờ). Người dùng sẽ đính kèm JWT cho mỗi lời gọi REST API sau đó. Thông thường JWT sẽ được đặt trong phần header authorization của mỗi yêu cầu. Tiến trình này có thể được minh họa bằng sơ đồ dưới đây.

Một JWT gồm 3 thành phần: header, payload và signature cách nhau bởi dấu chấm. Các thành phần này đều được biểu diễn dưới dạng (encode) base64, ví dụ:

Server này cung cấp REST API tương tự server các bạn đang dùng (từ buổi 2), tuy nhiên có hai điểm khác biệt chính sau:

  • Để thực hiện gọi REST API, client cần có header authorization chứa một JWT token hợp lệ.
  • Hỗ trợ 2 route mới là POST /api/auth/signup, POST /api/auth/signin lần lượt thực hiện chức năng đăng ký và đăng nhập.

POST /api/auth/signup: thực hiện đăng ký người dùng mới, client cần gửi dữ liệu JSON có dạng sau về server, ví dụ:

Nếu đăng ký thành công, người dùng có thể đăng nhập với tài khoản vừa tạo.

POST /api/auth/signin: thực hiện đăng nhập, client cần gửi dữ liệu JSON có dạng sau về server, ví dụ:

xxxxxxxxxx
{ "username": "baobui", "password": "123456" }

Nếu đăng nhập thành công, server sẽ gửi về một JWT có thời hạn trong vòng 24 giờ (bên trong JWT này có chứa id của người dùng).

Hãy dùng một REST API Client (ví dụ Postman) để thực hiện đăng nhập với người dùng vừa tạo (lưu ý đặt header Content-Type là application/json cho yêu cầu). Lưu giá trị của JWT trả về từ server. Tiếp theo, thực hiện yêu cầu GET /api/contacts với header authorization dạng như sau:

xxxxxxxxxx

Chụp hình kết quả thực hiện.

Bước 1: Tạo store vuex để lưu trữ dữ liệu/trạng thái đăng nhập của người dùng

Đối với các dữ liệu/trạng thái được sử dụng bởi nhiều component như dữ liệu/trạng thái người dùng đăng nhập thì nên tạo store vuex cho các dữ liệu/trạng thái này để dễ dàng quản lý chúng.

Các thành phần của module auth: state, mutations, actions, getters có thể được định nghĩa lần lượt như sau:

xxxxxxxxxx
const state = {
    user: {
        id: String,
        username: String,
        email: String,
        accessToken: String
    },
    status: {
        loggedIn: Boolean
    }
};
xxxxxxxxxx
const mutations = {
    initAuthState(state) {
        state.user = JSON.parse(localStorage.getItem("user"));
        state.status.loggedIn = !!state.user;
    },
    login(state, user) {
        state.status.loggedIn = true;
        state.user = user;
        localStorage.setItem("user", JSON.stringify(user));
    },
    logout(state) {
        state.status.loggedIn = false;
        state.user = null;
        localStorage.removeItem("user");
    },
    clearLoginStatus(state) {
        state.status.loggedIn = false;
    },
};
xxxxxxxxxx
const actions = {
    async login({ commit }, user) {
        let [error, response] = await handle(
            http.post("/auth/signin", {
                username: user.username,
                password: user.password,
            })
        );


        if (error || ! response.data.accessToken) {
            commit("logout");
            if (! error) {
                error = new Error("Whoops, no access token found!");
            }
            throw error;
        }


        commit("login", response.data);
        return response.data;
    },
    async register({ commit }, user) {
        const [error, response] = await handle(
            http.post("/auth/signup", {
                username: user.username,
                email: user.email,
                password: user.password,
            })
        );
        
        commit("clearLoginStatus");
        
        if (error) {
            throw error;
        }
        return response.data;
    }
};
xxxxxxxxxx
const getters = {
    userLoggedIn(state) {
        return state.status.loggedIn;
    },
    loggedInUser(state) {
        return state.user;
    }
};

Các action của store thực hiện các lời gọi REST API đến server để thực hiện đăng nhập, đăng ký. Trong trường hợp đăng nhập thành công, sẽ lưu thông tin người dùng ở trình duyệt nhờ vào localStorage sẵn có và đồng thời cập nhật trạng thái của store. localStorage cho phép lưu trữ dữ liệu dưới dạng các cặp khóa/giá trị. Khi đăng xuất thì thông tin người dùng sẽ được loại bỏ khỏi localStorage.

Tạo store và đăng ký vào ứng dụng: lần lượt tạo mới / thay đổi nội dung tập tin src/store/index.js và src/main.js:

Bước 3: Sử dụng vuex store cho các trang đăng nhập, đăng ký, profile

Hiệu chỉnh src/App.vue để khởi tạo trạng thái store khi chạy ứng dụng và lấy thông tin người dùng từ store:

xxxxxxxxxx
<script>
import { mapGetters, mapMutations } from "vuex";


export default {
    name: "App",
    computed: {
        ...mapGetters({
            currentUser: "loggedInUser",
        }),
    },
    methods: {
        ...mapMutations([
            "initAuthState" // map this.initAuthState() to this.$store.commit("initAuthState")
        ]),


        logout() {
            this.$store.commit("logout");
            this.$router.push("login");
        },
    },
    mounted() {
        this.initAuthState();
    }
};
</script>

Để ngăn không cho người dùng truy xuất các trang của ứng dụng ngoài đăng nhập (/login), đăng ký (/register) nếu chưa đăng nhập, ta có thể sử dụng guard beforeEach của router (đoạn mã trong beforeEach sẽ được thực thi trước khi đi ứng dụng chuyển sang một route mới). Hiệu chỉnh src/router/index.js:

xxxxxxxxxx
import { createWebHistory, createRouter } from "vue-router";
import store from "../store";


const routes = [
    //...
];


const router = createRouter({
    history: createWebHistory(),
    routes,
});


router.beforeEach((to, from, next) => {
    const publicPages = ["/login", "/register"];
    const authRequired = !publicPages.includes(to.path);
    const loggedIn = store.getters.userLoggedIn;


    if (authRequired && !loggedIn) {
        next("/login");
    } else {
        next();
    }
});


export default router;

Bước 4: Thêm header authorization vào các yêu cầu thao tác trên tài nguyên (các liên hệ)

Hiệu chỉnh src/services/contact.service.js thêm header authorization vào mỗi yêu cầu REST API:

xxxxxxxxxx
import http from "../common/http";
import store from "../store";


class ContactService {
    constructor() {
        http.interceptors.request.use(config => {
            const user = store.getters.loggedInUser;
            if (user && user.accessToken) {
                config.headers.authorization = `Bearer ${user.accessToken}`;
            }
            return config;
        });     
    }
    getAll()            { return http.get("/contacts");  }
    get(id)             { return http.get(`/contacts/${id}`); }
    //...
}


export default new ContactService();
